Priority Scheduling

What is Priority Scheduling


Priority scheduling is a method of scheduling processes or tasks based on their priority. Each process or task is assigned a priority level, typically represented by a numerical value, where a higher number indicates a higher priority.


The scheduler selects the process with the highest priority to execute, and if multiple processes have the same priority, they are executed in a first-come, first-served (FCFS) manner.

Types of Priority Scheduling

___Priority scheduling divided into two main types___




Preemptive Scheduling:


In Preemptive Scheduling, the tasks are mostly assigned with their priorities. Sometimes it is important to run a task with a higher priority before another lower priority task, even if the lower priority task is still running. The lower priority task holds for some time and resumes when the higher priority task finishes its execution.


Non-Preemptive Scheduling:


In this type of scheduling method, the CPU has been allocated to a specific process. The process that keeps the CPU busy, will release the CPU either by switching context or terminating. It is the only method that can be used for various hardware platforms. That’s because it doesn’t need special hardware (for example, a timer) like preemptive scheduling.





Characteristics of Priority Scheduling


1. A CPU algorithm that schedules processes based on priority.

2. It used in Operating systems for performing batch processes.

3. If two jobs having the same priority are READY, it works on a FIRST COME, FIRST SERVED basis.

4. In priority scheduling, a number is assigned to each process that indicates its priority level.

5. Lower the number, higher is the priority.

6. In this type of scheduling algorithm, if a newer process arrives, that is having a higher priority than the currently running process, then the currently running process is preempted.





Advantages of priority scheduling

___Here, are benefits/pros of using priority scheduling method___




1. Easy to use scheduling method

2. Processes are executed on the basis of priority so high priority does not need to wait for long which saves time

3. This method provides a good mechanism where the relative important of each process may be precisely defined.

4. Suitable for applications with fluctuating time and resource requirements.





Disadvantages of priority scheduling

___Here, are cons/drawbacks of priority scheduling___




1. If the system eventually crashes, all low priority processes get lost.

2. If high priority processes take lots of CPU time, then the lower priority processes may starve and will be postponed for an indefinite time.

3. This scheduling algorithm may leave some low priority processes waiting indefinitely.

4. A process will be blocked when it is ready to run but has to wait for the CPU because some other process is running currently.

5. If a new higher priority process keeps on coming in the ready queue, then the process which is in the waiting state may need to wait for a long duration of time.

Algorithm for Priority Scheduling


Input:


1. List of processes, each with a unique process ID.

2. Priority value for each process, indicating its priority level (a lower number typically indicates a higher priority).

3. Burst time (or execution time) for each process.

4. Arrival time (optional if you want to handle arrival times).




Output:


1. Order in which processes are scheduled for execution.

2. Waiting time for each process.

3. Turnaround time for each process.

4. Average waiting time.

5. Average turnaround time.




Algorithm: (Steps)


1. Initialize data structures: >>Create an empty ready queue to hold processes. >>Create variables to track the total waiting time and turnaround time.

2. Add all processes to the ready queue, each with its priority and burst time.

3. Sort the ready queue based on the priority of each process. Processes with higher priority (lower priority values) should come first in the queue.

4. Initialize a variable current_time to keep track of the current time.

5. While the ready queue is not empty: >>Dequeue the process with the highest priority from the ready queue. This process has the highest priority value. >>Execute the selected process on the CPU.

6. If the system uses preemptive priority scheduling (where a higher-priority process can preempt a running process): >>Whenever a new process arrives in the ready queue with a higher priority or the priority of the currently running process increases, preempt the currently running process and replace it with the higher-priority process. The preempted process is placed back in the ready queue.

7. If the system uses non-preemptive priority scheduling (where processes run to completion without interruption): >>Allow the currently running process to complete its execution without interruption, even if higher-priority processes arrive in the ready queue during its execution.

8. For each completed process, calculate: >>Waiting time: The waiting time for a process is the total time it spent waiting in the ready queue before it was executed. You can calculate this as waiting_time = turnaround_time - burst_time.

9. Update the total waiting time and turnaround time by adding the values for the completed process.

10. Continue this process until all processes have been executed.

11. Calculate the average waiting time by dividing the total waiting time by the number of processes.

12. Calculate the average turnaround time by dividing the total turnaround time by the number of processes.

13. Once all processes are executed, generate the scheduling sequence and display the results, including the scheduling order, waiting times, and turnaround times.
